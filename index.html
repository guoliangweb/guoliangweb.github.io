一个堪比畅言评论系统的第三方插件快评

Posted on 2019-10-17

一个堪比畅言评论系统的快评评论系统

目前评论系统用的最多的应该就是畅言和Gitment了。接下来我们就分析下这两款产品的不同。

首先我们来说说畅言:

畅言是一个简单而强大的社会化评论及聚合平台。用户可以直接用自己的社会化网络账户在第三方网站发表评论，并且一键评论同步至社交网络将网站内容和自己的评论分享给好友。畅言算是做的很早的一款产品了, 但是他还是有很多不足的地方。

1, 畅言呢最让人头痛的一点就是需要备案, 当你注册看见表格里需要填写一大堆信息的时候就会头痛, 尤其看到有个备案号就更加抓狂。 其实备案这个事情, 也因为国家政策关系而不能避免的。正因为这一点, 对于那些想用 Hexo + github pages的用户没有备案号而隔在了门外。在这一点上无形之中就缺少一批用户。

2, 畅言在使用中最令人反感的就是那些堪称网站”牛皮癣”的广告了, 当你打开网站的时候, 下方出现一则关于医疗, 房产等杂七杂八的图片广告, 无形之中就拉低了网站的层次。即使你UI页面做的特别好看, 但用户体验极差, 也达不到好的效果。然后你又会想办法把广告去掉, 却又因高额的费用苦恼。

再来说说Gitment:

Gitment 是基于 GitHub Issues 的评论系统。支持在前端直接引入，不需要任何后端代码。可以在页面进行登录、查看、评论、点赞等操作，同时有完整的 Markdown / GFM 和代码高亮支持。尤为适合各种基于 GitHub Pages 的静态博客或项目页面。

1.优点：
Gitment支持在前端直接引入，不需要任何后端代码，可以在页面进行登录、查看、评论、点赞等操作，同时有完整的 Markdown / GFM 和代码高亮支持，尤为适合各种基于 GitHub Pages 的静态博客或项目页面。

GitHub的评论数据存放在GitHub的issue里，基本不用担心数据丢失或者GitHub关闭服务。

2.缺点：
只能使用 GitHub 账号进行评论。无形中过滤掉了一些想评论又没有账号却嫌麻烦的评论者。

每篇文章需要自己手动用GitHub账号登录并初始化，其他用户才能评论。那么问题来了, 如果每一篇文章都必须先初始化一遍才能使用，假如你有一百篇文章，你就需要手动初始化一百次！如果用脚本一键初始化，依然很麻烦。

在了解完这两款评论系统之后我果断选择了快评

什么是快评?它又有什么特色和吸引人的地方呢? 就让我们先了解一下快评吧!

其实快评它同样也是一款社会化的评论系统。 它是一款轻量级插件, 当看到的第一感觉就是简洁大方, 加载速度快, 使用方便, 登录方式也比较灵活, 目前支持微信和手机号登录。

快评的具体优势在哪

与畅言相比最大的特色就是可以使用Markdown编辑器进行评论回复, 这点深受一些重度Markdown患者和资深程序员的喜爱。

它还有一大特色就是评论系统无广告, 为用户带来更好的体验。

那相比Gitment来说, 快评使用微信和手机号登录更方便更快捷, 解决注册github带来的困扰, 大大缩短时间提升效率。

快评在评论时不像Gitment评论时那样繁琐, 不会每次都初始化。我们直接就可以进行评论回复, 也不用去安装脚本解决一键初始化问题, 这样可以减少用户花费时间做繁琐的工作, 从而进一步提升用户体验

评论系统 优点 缺点
畅言 功能上比较完善 备案, 高额广告费
Gitment Markdown语法 登录不灵活, 初始化操作繁琐
快评 Markdown语法, 界面简洁大方, 加载速度块, 登录灵活, 无广告 基本功能实现, 可满足广大用户需求, 后续功能在持续开发中

分析了这么多, 如果你还在纠结使用哪种产品, 可以先来尝试体验下效果, 我想它会给你带来意想不到的惊喜。
